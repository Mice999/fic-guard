# Incident Response：如果发现作品被搬运了，先做什么

按这个顺序来，不要颠倒。

## 第 0 步：先深呼吸，不要立刻在公开平台发声

很多创作者发现作品被搬之后的第一反应是：在自己的社交账号发"我作品被偷了，是 XXX"。这种反应很正常，但**它通常会让你后续的处境变更糟**：

- 把对方的注意力直接拉到你身上；
- 在你还没固定证据的时候，可能反过来被对方索赔；
- 对方有时间删除证据。

所以第一步是**取证，而不是发声**。

## 第 1 步：固定证据

按以下顺序操作：

1. **保存网页快照**。用 `Ctrl+S` 完整保存 HTML，或用 `archive.org` / `archive.today` 抓一份永久副本。后者特别重要——你的本地保存可以被对方质疑真伪，第三方归档不可以。
2. **截图**。整页截图（浏览器都有"完整页面截图"功能），包含 URL 栏、时间。
3. **下载侵权内容文本**。如果是 PDF/EPUB，把文件保留。
4. **用 fic-guard 验证**：
   ```bash
   fic-guard fingerprint extract 侵权文件.txt   # 如果你嵌过水印，能直接出来
   ```
   如果你之前生成过原作的指纹 / 时间戳，把它们和侵权文件放在一起。

## 第 2 步：确认你的原创证据链

把以下东西整理到一个文件夹：

- 原作文本（你自己持有的最早版本）；
- `*.fingerprint.json`（如果做过）；
- `*.proof.json`（如果做过）；
- `*.proof.json.ots`（如果 OTS stamp 过）；
- 你在原发布平台的发布时间截图。

证据链的核心问题是："你能证明你在对方之前持有这份内容吗？"上面这套材料能回答这个问题。

## 第 3 步：选择行动路径

到这一步你才决定要不要、向谁、怎么发起行动。常见路径有三条，可以并行：

### 路径 A：平台投诉（最快）

向**侵权内容所在平台**投诉，而不是向 AO3 投诉。绝大多数平台都有版权投诉入口（即使是国内平台）：

- 国内平台：通常在"用户协议"或"版权声明"页底部有邮箱
- 海外平台：DMCA takedown，几乎每个平台都有标准流程

需要提供：原作链接 + 你的身份证明（你不一定要用真实姓名，但至少要能证明你是原作账号的持有者）+ 证据。

### 路径 B：联系原发布平台

如果侵权方还在通过 AO3 / 你原发布的平台引流（比如以"AO3 资源整合"的名义建群），可以举报到原平台。AO3 自己也接受 ToS 违反报告。

### 路径 C：法律行动

只有在金额损失明显、对方在国内有可追溯主体的情况下，才值得考虑。同人作品本身的版权归属比较复杂（与原作 IP 持有方的关系），但**对你笔名/原创部分的盗用**通常是可追究的。咨询律师之前先把第 1、2 步做完，律师会感谢你。

## 第 4 步：再考虑要不要公开发声

发声有它的作用——让圈子里其他人警觉，给对方造成舆论压力。但发声**应该在你已经有完整证据链之后**，而不是之前。

发声时遵守几条：

- 描述事实，不下定性结论（"我的作品出现在 XX 群"而不是"XX 在偷作品"）；
- 不公布你认为是对方的真实身份信息——这一步一旦做了，你从受害者变成被告的概率会大幅上升；
- 留出投诉/法律渠道的优先级——舆论是补充，不是替代。

## 第 5 步：复盘

事情处理完之后，给自己一份复盘：

- 这次是怎么被抓到的？我下次该怎么减少暴露面？
- 我的备份、指纹、时间戳是不是都齐？
- 我的发布设置是不是该调整？

把答案写在你自己的笔记里，下一篇作品发布时跑一遍 `fic-guard safe-publish` 验证一遍。
