# Pattern: 镜像站 / 自动化转贴

## 模式描述

1. 攻击方搭建一个外观接近原站、或干脆是独立小说站的镜像。
2. 定时爬取目标站点内容，自动转载（往往保留作者笔名，但不附原作链接，或附一个失效链接）。
3. 通过 SEO 在搜索引擎获得流量，承接广告/会员变现。

## 识别信号

- 同样的作品出现在你没听说过的小说聚合站，且时间比你晚；
- 转载方"作者"字段是你的笔名，但联系方式 / 主页是空的或对方自己的；
- 站点广告密集，章节中间插弹窗；
- 你在原站点改了内容，几小时到几天后镜像站也跟着变（说明是自动同步）。

## 对创作者的影响

- 读者搜索时先到达镜像站，原作者收益 / 反馈被截胡；
- 镜像站可能不同步章节删除——你删了的章节继续挂着；
- 弹窗广告 / 不当内容 与你的署名绑定。

## 应对

**预防层**：

- `fic-guard fingerprint watermark` 在每章嵌入轻量水印；
- 文中偶尔放一些"诱饵句"——意思自洽但搜索唯一的句子，方便日后反查；
- 在原作末尾固定写一句"原发于 XX，转载请联系作者"——不是为了挡爬虫，是为了证据上明确"未授权"。

**事后层**：

- `fic-guard monitor` 跑你的指纹，能定位到这类站；
- 大部分这类站点在某地（CDN / 域名注册商 / 广告联盟）总有一个可投诉的环节。最有效的不是直接联系站长（他们不会回），而是举报到他们的广告联盟和域名注册商。
- 搜索引擎也接受"未授权转载"的去重申诉（Google 有 DMCA 表单，Bing 有相应渠道）。

## 局限

新镜像会层出不穷。把这当成"持续维护"而不是"一次性解决"，每季度跑一次 monitor。
